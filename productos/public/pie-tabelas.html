⚠ 3 descobertas importantes pra nao perder tempo
  • cost_center_name esta NULL em 100% dos hotel_orders. Pra agrupar por CC, cruzar hotel_orders.user_id → users.id → users.cost_center
  • into_travel_policy nao significa "dentro do limite". 99.98% e 1. Significa "empresa tem politica ativa". Pra saber se infringiu, comparar hotel_fare com o limite do JSON metadata da travel_policy.
  • Limites de politica estao em centavos no JSON. 18000 = R$ 180, 30000 = R$ 300. Campo metadata da travel_policy.

🔗 Como cruzar as tabelas — query de referencia

-- Query base: reservas de hotel com CC, nome do viajante e limite da politica SELECT h.protocol, u.name AS viajante, u.cost_center AS centro_custo, h.hotel_city, h.hotel_fare, h.num_nights, h.status, h.days_in_advance, h.purchase_date, -- Extrair limite de hotel do JSON da politica (em centavos → dividir por 100) SAFE_CAST(JSON_EXTRACT_SCALAR( JSON_EXTRACT(tp.metadata, '$.hotel_order.config[0]'), '$.limit' ) AS FLOAT64) / 100 AS limite_politica, -- Marcar se infringiu CASE WHEN h.hotel_fare > SAFE_CAST(JSON_EXTRACT_SCALAR( JSON_EXTRACT(tp.metadata, '$.hotel_order.config[0]'), '$.limit' ) AS FLOAT64) / 100 THEN 1 ELSE 0 END AS infringiu FROM `dw-onfly-dev.v3_gold.hotel_orders` h LEFT JOIN `dw-onfly-dev.views_onfly.users` u ON SAFE_CAST(h.user_id AS INT64) = u.id LEFT JOIN `dw-onfly-dev.views_onfly.travel_policy` tp ON SAFE_CAST(h.company_id AS INT64) = tp.company_id AND tp.status = 1 WHERE h.purchase_date >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 30 DAY) AND h.status = 'Emitted' AND h.company_id = @company_id
